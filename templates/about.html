About — Version {{ version }}

This is Version {{ version }} of Automating Dictionary Compilation (Hendo Projects): a {{ meta.get('dict_label','') }} dictionary whose entries are compiled offline from vetted sources and served read-only. The two machines are strictly separate: the compiler may consult databases, rule tables and a draft-gloss pivot; the lookup app you are using now can only retrieve what the compiler stored. No form, gloss, or transliteration is ever produced at query time.

SEDRA III (roots, lexemes, words, morphology, English glosses derived from Payne Smith, and the BFBS Peshitta NT concordance) — © 1996 George A. Kiraz, free for personal and academic use. Required acknowledgment appears in the footer of every page; see also G. A. Kiraz, ‘Automatic Concordance Generation of Syriac Texts’, in VI Symposium Syriacum 1992, ed. R. Lavenant, Orientalia Christiana Analecta 247, Rome, 1994. Files vendored via the MIT-licensed sedrajs project.

SEDRA IV API (sedra.bethmardutho.org) — the source of the vocalised Madnhāyā (eastern) headwords, fetched politely and cached by compile.py fetch-vocalised. Until that command is run on a normal internet connection, entries show the pending badge.

Olam English→Malayalam dataset (olam.in), ODbL — “Contains data from Olam (olam.in), made available under the Open Database License.” Used only to draft Malayalam gloss candidates, each tagged with the English key that matched so the chain is auditable.

Transliteration tables — human-editable TSVs in tables/, DRAFT v0, deliberately unvetted; they await checking against the East Syriac ↔ Malayalam convention in Joju Jacob’s slides (Dukrana 2024). Values shown carry the draft badge until then.

Filling the gaps

python compile.py fetch-vocalised   # eastern/western Unicode from SEDRA IV
python compile.py build --top 300   # widen coverage, rerun anytime

Quranic Arabic Corpus morphology 0.4 (© 2011 Kais Dukes, GNU GPL) — headwords, lemmas, roots, and morphology all come from here, already fully vocalised in Uthmani orthography (no fetch step needed, unlike the Syriac side). See corpus.quran.com. Underlying text: Tanzil Uthmani v1.0.2 (© Tanzil.info, CC BY-ND 3.0).

camel_morph MSA v1.0 (MIT, © 2018–2022 NYU Abu Dhabi) — English glosses, cross-matched from this independent MSA lexicon against each QAC lemma (exact or orthographically-folded match). This is a weaker chain than a lexicon's own native glosses, so every match carries the cross-matched badge, never source, and records exactly which camel_morph lexeme it hit — see the "Sources & confidence" panel on any entry with a gloss.
